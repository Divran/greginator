Step 1

Build the LCR.

Note: While this tutorial focuses on the LCR / MCR, this setup will work for any machine that has recipes containing fluids.

Note: The reason this tutorial only works for GTNH version 2.3.0+ is because that is when blocking mode was fixed for fluid recipes. Prior to this version, blocking mode would be completely ignored when the recipe contained fluids.

Step 2

Add a block form dual interface for the input.
NOTE:

  • The input must use a block interface.
  • You can optionally use an ME output hatch instead of multiple normal hatches.
  • Also put an output bus or ME output bus anywhere (not pictured).
  • I recommend placing the input bus in one of the edge casings, and not the center, because otherwise the block dual interface will block it from view and prevent you from accessing it

Step 3

  • Add fluid p2ps to each input hatch.
  • Each input hatch must be connected to its own separate fluid p2p linking a different face of the dual interface.
  • If a recipe requires more than four fluids, replace one of the input hatches with a quadruple input hatch.

Image showing the p2p connections

Step 4

Be sure to remember to enable blocking mode

Step 5

How do you scale this to lategame? Below are a few pictures from the setups I use for MCRs. It's essentially the same setup, just made for recipes multipled more.

NOTE: To make these setups fast, follow all the steps below:
  • Blocking mode is enabled in each of these setups to completely avoid all recipe conflicts.
  • Due to blocking mode being enabled, there is a chance the MCR will stop running for 5s when AE isn't able to insert items fast enough. To prevent this, make each recipe a multiple of 256 (the parallel of MCR). This works for any other machine, too, except you'd need to adjust the multiple by whatever that machine's parallel is.
  • If you run out of recipe slots, you can try making another set of buses and hatches on the other side of the MCR, but to avoid recipe conflicts it's safer to make a new MCR
